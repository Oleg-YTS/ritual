# 📦 Итог по проекту: MorgueBot

## 🎯 Назначение
Telegram-бот для автоматизации учёта работы двух моргов (**Первомайская 13** и **Мира 11**) и оформления ритуальных услуг.

## 🛠 Стек и Архитектура
* **Язык/Фреймворк:** Python 3 + `aiogram 3.x`
* **Сервер:** Render.com (Webhook режим)
* **Хранение:** Локальные JSON-файлы (`data/`)
* **Структура:** Модульная (`handlers/`, `database/`, `keyboards/`, `utils/`)

## 👥 Ролевая модель
| Роль | Возможности |
|------|-------------|
| **Агент** | ➕ Тела, 🗑️ Удаление, ⚰️/🔥 Заказы, 📋 Карточки |
| **Менеджер** | Всё выше + 🔒 Закрытие смен, 📈 Статистика по моргу |
| **Админ** | Полный доступ + 📊 Отчёты за период (оба морга) |

## ✅ Ключевой функционал
1. **Учёт тел:** Циклическое добавление, пометка оплаты, фиксация организаций вывоза/БСМЭ.
2. **Ритуальные заказы:** Похороны и кремация. Авто-генерация текстовых карточек для **водителя** и **крематория**.
3. **Финансы:** Автоматический расчёт дохода, расходов (санитары/перевозка/зарплата агента) и чистой прибыли.
4. **Планировщик:** Автозакрытие смен в 15:30, напоминания менеджерам (14:30, 15:00, 15:20).
5. **Отчётность:** Статистика по последней закрытой смене и сводные отчёты за неделю/месяц/квартал.

## 🌐 Деплой и поддержка
* Развёрнут на **Render.com**.
* Webhook установлен и работает.
* Настроен **Keep-Alive** (внешний пинг каждые 15 мин) для предотвращения засыпания.
* Код полностью очищен от тестовых и лишних файлов, запушен в ветку `main`.
