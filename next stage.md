# Первый этап развития: Группа ритуальных услуг (Тула)

## Цель
Создать простую, надёжную систему для **одной группы**:
- **Агент** — **только принимает заказ** и вводит данные  
- **Главный агент** — **единственный, кто распределяет задачи**, включая:
  - Оформление документов
  - Перевозку из места в морг (если нужно)
  - Перевозку между моргами (промежуточная)
  - Перевозку к кладбищу/храму

> ✅ **Всё внутри группы — никакого аутсорса**  
> ✅ **Агент не распределяет — он только фиксирует**  
> ✅ **Главный агент — логист, который знает, кто где и когда**

---

## 🔧 Техническая реализация

### 1. **Роли (`users.json`) — три роли**
```json
{
  "id": "12345",
  "username": "agent_ivanov",
  "role": "agent",           // только принимает заказ
  "group_id": "TULA-01"
}
```

> ✅ Только три роли:
> - `agent` — **только принимает заказ** — **никаких задач не получает**
> - `driver` — везёт тело (перевозка)  
> - `group_lead` — главный агент: видит все заказы, распределяет всё

> ⚠️ Все — **в одной группе** (`TULA-01`)  
> Никаких других групп, нет аутсорса, нет внешних сервисов

### 2. **Поля в заказе (`order_storage.py`)**
```python
"status": "created",               // created → assigned → done
"group_id": "TULA-01",
"agent": "agent_ivanov",           // кто принял заказ
"event_date": "25.04.2026",        // дата похорон/кремации
"temple_time": "14:00",            // ⭐ ВРЕМЯ ОТПЕВАНИЯ — ключевое!
"current_location": "Морг Первомайская 13",  // где тело сейчас
"current_type": "morgue" or "home", // тип места: морг или дом
"funeral_location": "Троекуровское",
"funeral_type": "burial" or "cremation",
"documents": {
  "death_cert": false,             // Справка о смерти
  "zags_cert": false,              // Свидетельство ЗАГС
  "burial_permit": false           // Разрешение на захоронение
},
"assigned_doc": "",                // кто получает документы (агент)
"assigned_pickup": "",             // кто везёт от current_location → next_location
"assigned_transfer": "",           // ⭐ НОВОЕ: кто везёт из морга в морг
"assigned_funeral": "",            // кто везёт на кладбище/храм
"doc_status": "pending",           // pending → completed
"pickup_status": "pending",        // pending | skipped | completed
"transfer_status": "pending",      // ⭐ НОВОЕ: pending | skipped | completed
"funeral_status": "pending"        // pending → completed
```

> ✅ **Ключевые изменения**:
> - `current_location` — **где тело сейчас** (дом или морг)  
> - `current_type` — `home` или `morgue`  
> - `assigned_pickup` — если `current_type: home` → везёт в морг  
> - `assigned_transfer` — **если `current_type: morgue` и нужно перевезти в другой морг**  
> - `assigned_funeral` — везёт из **последнего морга** на кладбище

> ✅ **Статусы теперь 3 состояния**:
> - `pending` — не начато  
> - `skipped` — не нужно  
> - `completed` — сделано

> ✅ **Промежуточная перевозка — это отдельная задача**, которую **только главный агент** может назначить.

### 3. **Интерфейс: `📋 Заказы`**
- **Только главный агент** видит кнопку: `📋 Заказы`  
- Показывает список всех заказов в группе:  
  ```
  📋 ЗАКАЗЫ:
  ⚰️ Петров — 25.04.2026 (от: Морг Первомайская 13)  
     📄 pending  🚗 pending  🚛 pending  ⚰️ pending
  🔥 Сидоров — 26.04.2026 (от: Дом: ул. Ленина, 15)  
     📄 pending  🚗 pending  🚛 skipped  ⚰️ pending
  ```

> ✅ Главный агент нажимает на заказ → открывает детали

### 4. **Распределение задач — строго по сценариям**

#### 📄 Назначить документы
> **"Кому передать документы?"**  
> ↓  
> Выпадающий список:  
> - `Иванов` (агент)  
> - `Петров` (агент)  
> - `Сидоров` (агент)  
>  
> → Главный агент выбирает: `Петров`  
> → Система отправляет:  
> `📄 Задание: Петров, Морг Первомайская 13 🔥 22.04`  
> **и ставит `doc_status: pending`**

> ✅ **Агент видит чек-лист:**  
> ```
> 📄 Документы:
> [ ] Справка о смерти
> [ ] Свидетельство ЗАГС
> [ ] Разрешение на захоронение
> ```
> → Ставит ✅, когда всё готово → `doc_status: completed`

#### 🚗 Назначить перевозку (от места в морг)
> **"Кто везёт тело в морг?"**  
> ↓  
> Выпадающий список:  
> - `Водитель 1`  
> - `Водитель 2`  
>  
> → **Только если `current_type: home`** — кнопка **активна**  
> → Если `current_type: morgue` — кнопка **не отображается**  
>  
> → Главный агент выбирает: `Водитель 1`  
> → Система отправляет:  
> `🚗 Задание: Петров, Дом: ул. Ленина, 15 → Морг Первомайская 13`  
> **и ставит `pickup_status: pending`**

> ✅ **Водитель подтверждает**:  
> `✅ Взял тело из: Дом: ул. Ленина, 15`  
> `✅ Привёз в: Морг Первомайская 13`  
> → `pickup_status: completed`

#### 🚛 Назначить промежуточную перевозку (морг → морг)
> **"Кто везёт тело из одного морга в другой?"**  
> ↓  
> Выпадающий список:  
> - `Водитель 1`  
> - `Водитель 2`  
>  
> → **Только если `current_type: morgue`** — кнопка **активна**  
> → **И только если главный агент знает**, что нужно перевезти в другой морг  
>  
> → Главный агент выбирает: `Водитель 2`  
> → Система отправляет:  
> `🚛 Задание: Петров, Морг Первомайская 13 → Морг Мира 11`  
> **и ставит `transfer_status: pending`**

> ✅ **Водитель подтверждает**:  
> `✅ Взял тело из: Морг Первомайская 13`  
> `✅ Привёз в: Морг Мира 11`  
> → `transfer_status: completed`

#### ⚰️ Назначить похороны/кремацию
> **"Кто везёт на похороны?"**  
> ↓  
> Выпадающий список:  
> - `Водитель 1`  
> - `Водитель 2`  
>  
> → **Всегда активна** — потому что **всегда нужна** финальная перевозка  
>  
> → Главный агент выбирает: `Водитель 1`  
> → Система отправляет:  
> ```
> 🚗 Водителю
> _________________________
> 📍 Морг: Мира 11
> 📅 Дата: 25.04.2026
> ⏰ Отпевание: 14:00
> 👤 Заказчик: Иванов И.И.
> 📞 Телефон: +79991234567
> 💀 Усопший: Петров П.П.
> 🪦 Гроб (размер): 175x65, стандарт
> ⛪ Храм: Храм Святого Николая
> 🪦 Кладбище: Троекуровское
> ```
> **и ставит `funeral_status: pending`**

> ✅ **Водитель подтверждает**:  
> `✅ Доставил тело на кладбище`  
> → Система меняет `funeral_status: completed`

## 💡 Почему так?

| Проблема | Решение |
|---------|---------|
| Агент должен только принимать заказ | Он не распределяет — только фиксирует |
| Промежуточная перевозка — частая практика | Отдельная кнопка `🚛` — только для морг → морг |
| Нет путаницы в логистике | Чёткие статусы: `skipped`, `pending`, `completed` |
| Главный агент — логист | Он видит все шаги и управляет ими |
