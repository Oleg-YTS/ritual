# Telegram-бот для учёта морга и ритуальных услуг

Бот для учёта работы морга (тела, смены, оплаты, расчёт зарплат) и оформления ритуальных заказов с сохранением данных в GitHub.

## Функционал

- **Учёт работы морга** — добавление тел, отметка оплаты, расчёт зарплат санитаров и перевозчиков
- **Ритуальные заказы** — оформление заказов с формированием маршрута для водителя
- **Отчёты** — статистика за неделю по моргу и ритуальным услугам
- **Хранение данных** — все данные сохраняются в GitHub (CSV файлы)

## Технологии

- Python 3.10+
- aiogram 3.4+
- PyGithub 2.1+
- python-dotenv 1.0+

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <your-repo-url>
cd ritual
```

### 2. Создать виртуальное окружение

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Mac/Linux:**
```bash
python -m venv venv
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните файл `.env`:

```env
# Telegram (получить у @BotFather)
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# GitHub (Settings → Developer settings → Personal access tokens → Fine-grained token)
GITHUB_TOKEN=github_pat_11ABCDEFG1234567890
REPO_NAME=username/morg-bot

# Опционально (если нужен авторежим MAX)
MAX_ENABLED=False
```

### 5. Запустить бота

```bash
python bot.py
```

## Получение токенов

### Telegram Bot Token

1. Откройте @BotFather в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям
4. Скопируйте полученный токен в `.env`

### GitHub Token

1. Откройте GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. Создайте новый токен с разрешениями:
   - Contents: Read and Write
   - Работает только с выбранными репозиториями
3. Скопируйте токен в `.env`
4. Убедитесь, что репозиторий существует и указан в `REPO_NAME`

## Структура данных в GitHub

После работы бота в репозитории создаётся следующая структура:

```
morg/
└── YYYY-MM/
    └── YYYY-MM-DD.csv      # Данные смен морга

ritual/
└── orders.csv              # Ритуальные заказы
```

### Формат CSV морга

```csv
Фамилия,Тип,Источник,Оплачено,Организация
Иванов,Стандарт,Отделение,да,
Петров,Не стандарт,Амбулаторно,нет,ООО Ритуал
```

### Формат CSV ритуальных заказов

```csv
Дата,Заказчик,Телефон,Усопший_адрес,Гроб,Храм,Кладбище,Зарплата_агента
2026-04-07 14:30,Иванов Иван Иванович,89123456789,Петров Петр ул.Ленина 5,Дубовый лакированный,Храм Всех Святых,Северное,3000
```

## Команды бота

| Команда/Кнопка | Действие |
|----------------|----------|
| `/start` | Показать главное меню |
| `➕ Новая смена` | Начать смену в морге |
| `➕ Добавить тело` | Добавить ещё одно тело (во время смены) |
| `🔒 Закрыть смену` | Завершить смену, расчёт, сохранение |
| `🕯️ Ритуальный заказ` | Оформить новый заказ |
| `📊 Отчёт за неделю` | Показать статистику за 7 дней |
| `📋 Список заказов` | Показать последние 10 заказов |

## Расчёт зарплат

### Смена морга

- **Доход** = (кол-во Стандарт опл × 8000) + (кол-во Не стандарт опл × 10000)
- **Зарплата санитаров** = (кол-во Стандарт опл × 6500) + (кол-во Не стандарт опл × 8000)
- **Зарплата перевозчика** = кол-во опл всего × 1500
- **Расход** = Зарплата санитаров + Зарплата перевозчика
- **Прибыль** = Доход − Расход

## Деплой на Render.com

1. Создайте аккаунт на [render.com](https://render.com)
2. Нажмите "New +" → "Web Service"
3. Подключите ваш GitHub репозиторий
4. Настройки:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Environment Variables**: добавьте `BOT_TOKEN`, `GITHUB_TOKEN`, `REPO_NAME`
5. Нажмите "Create Web Service"

## Важные замечания

- Файл `.env` **НЕ** должен быть загружен в GitHub (добавлен в `.gitignore`)
- Бот использует FSM (Finite State Machine) для пошаговых сценариев
- При проблемах с GitHub данные сохраняются локально как fallback
