# MorgueBot — Telegram-бот для учёта моргов и ритуальных услуг

Бот для управления двумя моргами (Первомайская 13, Мира 11), учёта тел, смен, ритуальных заказов и формирования отчётов.

## 🔐 ВАЖНО: Настройка GitHub

Бот работает **только с GitHub** — все данные хранятся в репозитории.

### Получение токена GitHub:
1. Перейдите в [GitHub → Settings → Developer settings → Personal access tokens](https://github.com/settings/tokens)
2. Создайте токен с правами `repo`
3. Скопируйте его и вставьте в `.env` как `GITHUB_TOKEN`
4. Убедитесь, что `GITHUB_REPO=Oleg-YTS/ritual`

**Бот не запустится без этого токена.**

## 🔄 Как это работает

- Все данные (заказы, смены, пользователи) хранятся в файлах репозитория
- При запуске бот загружает актуальные данные из GitHub
- При каждом изменении (заказ, добавление тела) — данные сразу записываются в GitHub
- Никаких локальных файлов — всё в облаке
- Вы всегда видите актуальные данные в GitHub

## 📁 Структура данных

- `data/morgue1.json` — данные Первомайского морга
- `data/morgue2.json` — данные морга на Мира 11
- `data/users.json` — список пользователей и их роли

## 📊 Автоматическое закрытие смен

Смены автоматически закрываются в 00:00, если не были закрыты вручную.

## Функционал

### Управление моргами
- **Два морга**: Первомайская 13 и Мира 11
- **Смены**: автоматическое открытие при первом действии, ручное/автоматическое закрытие
- **Учёт тел**: добавление, удаление (БСМЭ), отметка оплаты
- **Финансы**: автоматический расчёт дохода, расходов и прибыли

### Ритуальные услуги
- **Заказы**: похороны и кремация с полным набором параметров
- **Карточки**: автоматическое формирование для водителя и крематория
- **Маршруты**: учёт доп. услуг (зал, отпевание, срочная кремация)

### Отчёты и аналитика
- **Статистика смены**: мгновенный расчёт при закрытии
- **Отчёты за период**: неделя, месяц, квартал
- **Аналитика «кто вывез»**: список удалённых тел с организациями
- **Зарплата агента**: учёт по каждому заказу

### Роли и права
| Роль | Описание |
|------|----------|
| `admin` | Полный доступ ко всем моргам и функциям |
| `manager_morg1` | Менеджер Первомайская 13 |
| `manager_morg2` | Менеджер Мира 11 |
| `agent_morg1` | Агент Первомайская 13 |
| `agent_morg2` | Агент Мира 11 |

## Технологии

- Python 3.10+
- aiogram 3.x
- FastAPI + aiohttp (webhook)
- JSON-хранилище (файлы в папке `data/`)
- FSM (Finite State Machine) для многошаговых действий

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <your-repo-url>
cd ritual
```

### 2. Создать виртуальное окружение

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Mac/Linux:**
```bash
python -m venv venv
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения

Создайте файл `.env` на основе `.env.example`:

```bash
copy .env.example .env
```

Заполните файл `.env`:

```env
# Telegram (получить у @BotFather)
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# Webhook (опционально)
WEBHOOK_PATH=/webhook
WEBHOOK_SECRET=your_secret_token

# Для продакшена (Render)
RENDER_EXTERNAL_URL=https://your-app.onrender.com
PORT=10000
```

### 5. Запустить бота

**Локальная разработка (polling):**
```bash
python bot.py --polling
```

**Продакшен (webhook на Render.com):**
```bash
python bot.py
```

## Получение Telegram Bot Token

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для выбора имени и username
4. Скопируйте полученный токен в `.env`

## Деплой на Render.com

### 1. Подготовка

1. Создайте аккаунт на [render.com](https://render.com)
2. Форкните репозиторий в GitHub

### 2. Создание сервиса

1. В панели Render нажмите **"New +"** → **"Web Service"**
2. Подключите ваш GitHub репозиторий
3. Настройте параметры:
   - **Name**: `morgue-bot` (или любое другое)
   - **Region**: `Frankfurt` (ближайший к вам)
   - **Branch**: `main`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Plan**: `Free`

### 3. Переменные окружения

Добавьте следующие переменные в разделе **Environment**:

| Ключ | Значение |
|------|----------|
| `BOT_TOKEN` | Токен от @BotFather |
| `WEBHOOK_SECRET` | Любой секретный токен (например, `supersecret123`) |

**Важно**: Render автоматически установит:
- `RENDER_EXTERNAL_URL` — URL вашего сервиса
- `PORT` — порт (обычно 10000)

### 4. Настройка webhook

После деплоя бот автоматически установит webhook при первом запуске.

Проверить можно через:
```
https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo
```

### 5. Файл Procfile

Уже создан в репозитории:
```
web: python bot.py
```

## Важные замечания

1. **Файл `.env` НЕ должен быть в GitHub** (добавлен в `.gitignore`)
2. **Автоматическое закрытие смены**: если смена не закрыта до 00:00, все тела считаются оплаченными
3. **FSM**: бот использует конечный автомат состояний для многошаговых действий
4. **JSON-хранилище**: все данные сохраняются локально в папке `data/`
5. **Логирование**: все действия логируются с временными метками
