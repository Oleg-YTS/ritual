# 🚀 План развития MorgueBot: Диспетчерская и Водители

## 📌 Концепция
Расширение функционала бота для внедрения логистического модуля. Позволит централизованно распределять ритуальные заказы между водителями, фиксировать время выезда и контролировать выполнение заданий.

## 👥 Новые роли и права доступа

| Роль | Описание | Доступное меню |
|------|----------|----------------|
| **Диспетчер** (`dispatcher`) | Координатор логистики. Видит заказы по обоим моргам, назначает водителей, указывает время сбора. | 📦 Распределение, 📊 Все заказы |
| **Водитель** (`driver`) | Исполнитель. Видит только назначенные ему задания, может подтвердить выполнение. | 🚕 Мои задания, 📞 Контакты |

## 🗃 Изменения в структуре данных

### 1. Расширение заказа (`data/morgue*.json`)
В структуру заказа добавляется блок `dispatch`:
```json
{
  "order_id": "...",
  "event_date": "12.04.2026",
  "deceased": "Иванов И.И.",
  "type": "funeral",
  "morgue_location": "Первомайская 13",
  "temple": "Храм Всех Святых",
  "dispatch": {
    "assigned_driver_id": 789123456,
    "assigned_driver_name": "Петров П.П.",
    "temple_time": "09:30",       // Время, когда водитель должен быть в храме
    "status": "assigned",         // pending -> assigned -> completed
    "assigned_at": "2026-04-12 08:15"
  }
}
```

### 2. Реестр пользователей (`data/users.json`)
Добавление новых записей:
```json
"789123456": { "role": "driver", "name": "Петров П.П.", "phone": "+79991234567" },
"987654321": { "role": "dispatcher", "name": "Сидоров А.А." }
```

## 🔄 Логика взаимодействия

### 👨‍💼 Диспетчер
1. Нажимает **📦 Распределение**.
2. Видит список заказов со статусом `pending` (по всем моргам).
   *Формат:* `⚰️ Иванов И.И. | 12.04 | Первомайская 13 -> Храм`
3. Выбирает заказ.
4. Бот просит: `⏰ Укажите время отпевания (ЧЧ:ММ), к которому водитель должен быть на месте:`
5. Вводит время (например, `09:30`).
6. Бот показывает список доступных водителей (`role: driver`).
7. Диспетчер выбирает водителя.
8. **Итог:** Заказ обновляется в БД, водителю приходит уведомление, диспетчеру подтверждение.

### 🚗 Водитель
1. Получает личное сообщение:
   `📋 НОВОЕ ЗАДАНИЕ\n⚰️ Похороны: Иванов И.И.\n📍 Морг: Первомайская 13\n⛪ Храм: Храм Всех Святых (быть к 09:30)\n👤 Контакт заказчика: +7...`
2. В главном меню нажимает **🚕 Мои задания**.
3. Видит свои активные заказы. Может открыть детали и нажать `✅ Выполнено` (меняет статус заказа в БД).

## ⚠️ Технические нюансы
1. **Telegram API:** Бот может писать водителям только если они ранее нажали `/start` или написали боту. Водители должны быть "активны" в боте.
2. **Конфликты записи:** При одновременном доступе диспетчеров к одному заказу нужна блокировка или проверка `if order.dispatch.status == "pending"`.
3. **Масштабируемость:** При росте >500 заказов JSON может начать тормозить. В будущем возможен переход на `SQLite` или `PostgreSQL`, но для текущего объема JSON справится.
